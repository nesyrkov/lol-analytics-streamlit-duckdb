# League of Legends Analytics

Интерактивный аналитический проект по игре **League of Legends**, построенный на связке **DuckDB + Streamlit**.

Проект доступен по ссылке: [League of Legends Analytics App](https://lol-analytics-app-duckdb-btztumpzxzcxhkkdckdeqg.streamlit.app/)

## О проекте

Этот проект предназначен для анализа матчей League of Legends: команд, игроков, чемпионов, результатов игр и игровых показателей. Данные загружаются и обрабатываются с помощью DuckDB, а пользовательский интерфейс реализован в Streamlit.

Главная цель проекта - сделать данные по матчам удобными для просмотра, фильтрации и первичного анализа без необходимости вручную работать с большими таблицами.

## Что такое League of Legends

**League of Legends** - это командная MOBA-игра от Riot Games, в которой две команды по пять игроков сражаются друг против друга на карте Summoner's Rift. Основная цель матча - разрушить главное строение соперника, Нексус.

Ключевые особенности игры:

- формат матча: 5 на 5;
- жанр: MOBA, Multiplayer Online Battle Arena;
- основная карта: Summoner's Rift;
- роли игроков: top, jungle, mid, bot/ADC, support;
- важные игровые объекты: башни, драконы, Барон Нашор, ингибиторы и Нексус;
- победа зависит от командной игры, выбора чемпионов, контроля карты и реализации преимущества.

## Герои / чемпионы

Игровые персонажи в League of Legends называются **чемпионами**. В игре более 160 чемпионов, и каждый из них имеет уникальные способности, роль и стиль игры.

Чемпионы могут выполнять разные функции:

- **танки** - принимают на себя урон и начинают сражения;
- **бойцы** - совмещают урон и выживаемость;
- **маги** - наносят магический урон способностями;
- **ассасины** - быстро устраняют ключевые цели соперника;
- **стрелки** - наносят стабильный урон с дистанции;
- **саппорты** - помогают команде контролем, лечением, защитой и обзором карты.

Выбор чемпионов сильно влияет на стратегию команды: одни составы сильнее в ранней игре, другие раскрываются ближе к поздним стадиям матча.

## Инфраструктура проекта

Проект построен на двух основных инструментах:

### DuckDB

**DuckDB** используется как аналитическая база данных. Она хорошо подходит для работы с табличными файлами и позволяет выполнять SQL-запросы прямо по данным без отдельного серверного хранилища.

В проекте DuckDB используется для:

- хранения и обработки матчевых данных;
- выполнения SQL-запросов;
- агрегации статистики;
- фильтрации по регионам, командам, игрокам и чемпионам;
- подготовки данных для отображения в интерфейсе.

### Streamlit

**Streamlit** используется для создания веб-интерфейса аналитического приложения.

В приложении можно:

- просматривать данные по матчам;
- анализировать команды и игроков;
- сравнивать игровые показатели;
- фильтровать данные;
- быстро получать таблицы и визуальные представления без написания кода.

## Основные данные

В проекте используются данные по матчам League of Legends. Они могут включать:

- регион проведения матча;
- информацию о командах;
- данные об игроках;
- выбранных чемпионов;
- результат матча;
- игровые показатели команд и игроков;
- статистику по убийствам, смертям, ассистам и другим метрикам;
- длительность матча;
- данные по игровым объектам и преимуществу команд.

Такая структура позволяет анализировать как отдельные матчи, так и общие закономерности по регионам, командам и чемпионам.

## Возможности аналитики

Приложение помогает отвечать на вопросы:

- какие команды чаще выигрывают;
- какие чемпионы используются чаще всего;
- какие игроки показывают лучшие статистические результаты;
- как различаются показатели по регионам;
- какие игровые метрики связаны с победой;
- какие паттерны можно увидеть в данных матчей.
